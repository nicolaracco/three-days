# DEVLOG — Three Days

## 2026-04-26 — Architectural foundation + Day 1 scaffold; Cloudflare reorg surprise

The architectural layer is locked: GDD v0.3, 10 ADRs (numbering stable from this point forward), three custom agents (spec-author, implementer, quality-reviewer), and spec 0001 Approved and implemented. The scaffold itself landed cleanly — Bun 1.3.13 + Vite 7 + Phaser 3.90 + strict TS + `bun test`, layered `src/` per ADR-0004, hello-world via `BootScene → MenuScene`, sanity test exercised the red-green-verify loop end to end. Three real corrections worth keeping. (1) **Bun's TypeScript story is half** — runtime runs TS natively, but Bun's docs are explicit that the bundler is not intended to replace `tsc` for typechecking, so `typescript` stays as a devDep; the easy "we don't need TypeScript" framing is wrong. (2) **Cloudflare reorganized Pages into Workers Builds mid-deploy** — the dashboard's "deploy command" field defaulting to `npx wrangler deploy` is the new flow, requiring a `wrangler.jsonc` at repo root and an ADR-0010 rewrite while the user was already mid-Save-and-Deploy; the *decision* (Cloudflare for previews) was unchanged but the *implementation* drifted under us. (3) **The post-edit hook fired typecheck on the first `.ts` write before `bun install` had run** and exit-2'd, blocking the workflow until I hardened it with a `node_modules` existence check; "bootstrap-friendly" is now the explicit contract, not a vague aspiration. Aside: `bun install` advertised Phaser `v4.0.0`, but per Phaser's own docs site that's rc.6 — stayed on 3.90 and recorded the choice in ADR-0001 Alternatives. **Next:** user does the CF Workers Builds dashboard click-through and confirms the iPhone preview, then spec 0002 (click-to-move with AP, GDD §13 Day 1's interactive deliverable).
